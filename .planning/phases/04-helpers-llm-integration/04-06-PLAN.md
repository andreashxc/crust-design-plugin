<objective>
Add explicit streaming LLM support over long-lived `chrome.runtime.Port` so long responses are not dropped by one-shot messaging or service-worker idle behavior.
</objective>

<tasks>

## Task 1: Port Protocol

Files:

- `apps/extension/src/background/llm/stream.ts`
- `apps/extension/entrypoints/background.ts`

Action:

- Register `chrome.runtime.onConnect` for port name `LLM_STREAM`.
- Define message shapes:
  - client start: experimentId, prompt, options
  - server delta
  - server done
  - server error
  - server partial_on_close
- Keep top-level listener registration synchronous.

Acceptance criteria:

- Unknown port names ignored.
- Missing key/config sends error and disconnects cleanly.

## Task 2: Provider Stream Normalization

Files:

- `apps/extension/src/background/llm/stream.ts`
- `apps/extension/src/background/llm/stream.test.ts`

Action:

- Normalize OpenAI and Anthropic streaming events into text deltas.
- Accumulate final text and usage if provider exposes it.
- Update session counters when stream completes or when usage becomes available.
- Cache final text on successful complete if cache enabled.

Acceptance criteria:

- Mock OpenAI stream emits deltas and done.
- Mock Anthropic stream emits deltas and done.
- Provider error emits structured error.

## Task 3: Content Helper Streaming API

Files:

- `apps/extension/src/content/helpers.ts`
- `apps/extension/src/content/helpers.test.ts`

Action:

- Support explicit streaming option without changing default `await llm(prompt)` string behavior.
- Provide a practical author API, e.g. async iterator or callback option.
- Abort signal closes the port.

Acceptance criteria:

- Default non-streaming helper still uses 04-03.
- Streaming helper receives deltas and final text.
- Port close before done returns/throws explicit partial result, not silent success.

## Task 4: Long Stream Simulation Test

Files:

- `apps/extension/src/background/llm/stream.test.ts`

Action:

- Simulate a stream longer than a normal one-shot message lifecycle.
- Verify port remains active until done or explicit disconnect.

Acceptance criteria:

- Test proves partial result is surfaced if port closes.
