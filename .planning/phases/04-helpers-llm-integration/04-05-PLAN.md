<objective>
Implement `fetchPage(url, selector?)` through the service worker and offscreen DOMParser with structured SPA-shell detection.
</objective>

<tasks>

## Task 1: Manifest Permission And Offscreen Entrypoint

Files:

- `apps/extension/wxt.config.ts`
- `apps/extension/entrypoints/offscreen.html`
- `apps/extension/entrypoints/offscreen.ts`

Action:

- Add `offscreen` permission.
- Add static offscreen document entrypoint for DOM parsing.
- Implement offscreen message handler for parse requests.

Acceptance criteria:

- Build manifest includes `offscreen`.
- Offscreen entrypoint is bundled.

## Task 2: Background Fetch Service

Files:

- `apps/extension/src/background/fetch-page.ts`
- `apps/extension/src/background/fetch-page.test.ts`

Action:

- Fetch only `http:`/`https:` URLs.
- Enforce timeout and max HTML size.
- Reject non-HTML content types.
- Ensure/create offscreen document before parsing.
- Return normalized `FetchPageResult`.

Acceptance criteria:

- Network error, non-HTML, too-large, selector-not-found, and success are tested.

## Task 3: SPA-Shell Detection

Files:

- `apps/extension/src/background/fetch-page.ts`
- `apps/extension/src/background/fetch-page.test.ts`
- `apps/extension/entrypoints/offscreen.ts`

Action:

- Detect likely SPA shell by low meaningful text, app-root-only body, high script-to-content ratio, or empty selector result.
- Return `{ ok:false, reason:'likely_spa_shell' }` for shell-like pages.

Acceptance criteria:

- SSR fixture returns parsed content.
- SPA-shell fixture returns structured warning, not empty success.

## Task 4: Typed Message And Content Helper Wiring

Files:

- `apps/extension/src/shared/messages.ts`
- `apps/extension/src/background/handlers/fetch-page.ts`
- `apps/extension/entrypoints/background.ts`
- `apps/extension/src/content/helpers.ts`

Action:

- Add `FETCH_PAGE` ProtocolMap entry.
- Register handler at top level.
- Wire content helper to background message.

Acceptance criteria:

- Content helper test proves request/response shape.
- Handler does not expose privileged internals.
