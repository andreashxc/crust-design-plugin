<objective>
Implement non-streaming `llm()` service-worker proxy for OpenAI and Anthropic, with cache, retry/backoff, normalized usage, provider tests, and session counters. Content scripts still cannot access API keys.
</objective>

<tasks>

## Task 2: Provider Abstraction

Files:

- `apps/extension/src/background/llm/providers.ts`
- `apps/extension/src/background/llm/service.test.ts`

Action:

- Implement normalized provider clients:
  - OpenAI complete via Responses API or REST equivalent.
  - Anthropic complete via Messages API or REST equivalent.
- Normalize to `{ text, provider, model, usage }`.
- Add `testProvider(provider)` using a minimal low-token request.

Acceptance criteria:

- Mocked provider responses produce normalized text and usage.
- Provider errors are normalized without leaking keys.

## Task 3: LLM Service: Cache, Retry, Counters

Files:

- `apps/extension/src/background/llm/service.ts`
- `apps/extension/src/background/llm/service.test.ts`

Action:

- Implement `completeLlm({ experimentId, prompt, options })`.
- Resolve provider/model/defaults from storage.
- Return cache hit if valid.
- Retry retryable failures with exponential backoff and max attempts.
- Increment session counters only for network calls.
- Store last LLM error on failure.

Acceptance criteria:

- Missing key returns structured error.
- Identical prompt within TTL returns cached result.
- 429/5xx retry up to max attempts.
- 401/config errors do not retry.
- Token counters increment from normalized usage.
