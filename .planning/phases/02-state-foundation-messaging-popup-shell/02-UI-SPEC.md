## Popup Outer Dimensions

Locked dimensions (browser-extension popup constraints — Chromium MV3 caps popups at ~800×600 max but smaller is courteous):

| Dimension | Value | Rationale |
|-----------|-------|-----------|
| Width | `360px` (fixed via `body { width: 360px }` in `popup.css`) | Wide enough to show experiment name + status Badge + Switch on one line; narrow enough to not crowd the toolbar dropdown. Replaces Phase 1's `min-width: 280px` (which was a development placeholder). |

## Spacing Scale

Declared values (multiples of 4 only — Tailwind v4 default scale: `1=4px, 2=8px, 3=12px, 4=16px, 6=24px, 8=32px`):

| Token | Tailwind class | Value | Usage in Phase 2 popup |
|-------|----------------|-------|------------------------|
| xs | `gap-1`, `mt-1`, `p-1` | 4px | name → description, description → switch row tight stack, icon→label gap inside trigger |
| sm | `gap-2`, `mt-2`, `p-2` | 8px | card → card gap, author trigger → first card gap, header → border tight |
| md | `gap-3`, `p-3` | 12px | body padding, card padding, author group → next author group |
| lg | `gap-4`, `mt-4`, `p-4` | 16px | (NOT USED in Phase 2 popup — reserved for Phase 3 tweak rows) |
| xl | `gap-6` | 24px | (NOT USED in Phase 2 — reserved for Phase 3 sectioning) |

**Exceptions:**
- Switch tap target height: `24px` (shadcn default `Switch` is `h-6 w-11` — meets WCAG 2.1 AA minimum 24×24 with `padding-y` from card body).
- Button (ghost variant) "Show stack trace" minimum height: `32px` (shadcn default `Button` `size="sm"` → `h-8 = 32px`; ensures comfortable click target on tightly-packed cards).
- Collapsible chevron + author label tap target: full row width (`w-full`), height `32px` minimum (matches Button sm) so the entire trigger row is clickable, not just the chevron icon.

shadcn's default `slate` palette + cssVariables. CSS variables are referenced via Tailwind v4 `bg-background`, `bg-card`, etc.

| Role | CSS variable (slate light) | CSS variable (slate dark) | Usage in Phase 2 popup |
|------|----------------------------|---------------------------|------------------------|
| Dominant 60% (background) | `--background: oklch(1 0 0)` (white) | `--background: oklch(0.129 0.042 264.695)` (slate-950) | Popup body background; empty space; default surface |
| Secondary 30% (cards) | `--card: oklch(1 0 0)` + `--border: oklch(0.929 0.013 255.508)` (slate-200) | `--card: oklch(0.208 0.042 265.755)` + `--border: oklch(1 0 0 / 10%)` | Per-experiment Card surfaces; group dividers |
| Accent 10% (applied state) | `--primary: oklch(0.208 0.042 265.755)` (slate-900) → Badge `default` variant | `--primary: oklch(0.984 0.003 247.858)` (slate-50) | Badge `variant="default"` for `applied` state ONLY. **Reserved for "this experiment is currently active on this tab" — the only positive-state signal in the popup.** |
| Destructive | `--destructive: oklch(0.577 0.245 27.325)` (red-600 family) | `--destructive: oklch(0.704 0.191 22.216)` (red-500 family) | Badge `variant="destructive"` for `error` and `auto-disabled` states; inline error text (`text-destructive`); auto-disable banner foreground |
| Muted (gray surfaces) | `--secondary: oklch(0.968 0.007 247.896)` (slate-100) / `--muted-foreground: oklch(0.554 0.046 257.417)` (slate-500) | dark equivalents | Badge `variant="secondary"` for `disabled` "Off" state; description text (`text-muted-foreground`); meta info |
| Outline (gray-outline) | `--border: oklch(0.929 0.013 255.508)` + transparent bg | dark equivalents | Badge `variant="outline"` for `pending` state |

**Accent (`primary` slate-900) is reserved for:**
1. Badge `variant="default"` on the `applied` status — and ONLY this badge in Phase 2.
2. Switch's checked-state thumb (shadcn `Switch` checked uses `bg-primary` automatically — this is part of the contract, not a separate accent).
3. Focus ring (`ring-ring` = `--ring: oklch(0.704 0.04 256.788)` in light mode) — also driven by primary in shadcn defaults.

**Accent NOT reserved for** (explicit anti-list to prevent UI sprawl):
- Author group labels (use `text-foreground` default, not primary)
- "Show stack trace" trigger (use `text-foreground` default, not primary)
- Description text (use `text-muted-foreground`, never primary)
- Header H1 "Experiments" (use `text-foreground` default)

**Destructive is reserved for** (the second semantic color):
1. Badge `variant="destructive"` on `error` state.
2. Badge `variant="destructive"` with `className="border border-destructive bg-transparent text-destructive"` for `auto-disabled` (red OUTLINE — visually distinct from solid red `error`).
3. Inline error message text (`text-destructive`).
4. Auto-disable banner text (`text-destructive font-medium`).
5. `AlertCircle` icon (12px) prefix on the auto-disable banner.

**Color-blind safety contract:** every Badge variant carries a TEXT label ("Off"/"Pending"/"Applied"/"Error"/"Auto-disabled"). Color is redundant with text. Verified against shadcn Badge defaults — the `variant` prop changes both color AND requires children text.

## Status Badge Contract (locked from CONTEXT D-23)

| State | shadcn Badge variant | Tailwind class override | Label text | When |
|-------|----------------------|-------------------------|------------|------|
| `disabled` | `secondary` | none | `Off` | Toggle is off (user intent = disabled) |
| `pending` | `outline` | none | `Pending` | Toggle on, but active tab URL doesn't match `scope.match` |
| `applied` | `default` | none | `Applied` | Toggle on, scope matches active tab, last apply succeeded — this is the SOLE green/primary affirmative state |
| `error` | `destructive` | none | `Error` | Toggle on, last apply or cleanup threw, error count below auto-disable threshold |
| `auto-disabled` | `destructive` | `className="bg-transparent text-destructive border-destructive border"` | `Auto-disabled` | Engine tripped circuit breaker (≥3 errors in 10s) |

**Implementation note for executor:** the auto-disabled variant is "destructive outline" — there is no built-in shadcn variant for this. Override solid red with the className above. Document this in the row component file as a comment so a future maintainer doesn't mistakenly switch to `variant="outline"` (which is gray, not red).

## Switch (Toggle) Behavior Contract (UI-04 + CONTEXT D-24)

| Property | Value |
|----------|-------|
| Component | shadcn `Switch` (Radix `@radix-ui/react-switch`) |
| Default size | shadcn default: `h-6 w-11` (24×44 px) — meets WCAG 2.1 AA tap target |
| Position | right-aligned in card; row is `flex justify-end` |
| `aria-label` | `"Toggle {experiment.name}"` — REQUIRED on every Switch |
| Keyboard | `Space` toggles (Radix default). `Tab` moves focus to next interactive element (next Switch or next collapsible trigger). |
| Focus ring | shadcn default `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2` — DO NOT override |
| Pending state (in-flight `runtime.sendMessage`) | `disabled={pending}` AND replace the thumb with a spinner: place a `<Loader2 className="size-3 animate-spin" />` absolutely-positioned over the thumb, OR adjacent to the Switch. Recommended placement: 4px to the LEFT of the Switch (`<Loader2 />` then `<Switch />` in a flex row; spinner only renders when `pending === true`). |
| On error response from SW | (a) snap Switch back to its previous value via local state (the source-of-truth `enabled` map will reconcile via `chrome.storage.onChanged`), (b) render an inline alert in the SAME card under the Switch row: `<p class="text-xs text-destructive mt-2">Toggle failed: {error}</p>`, (c) the inline alert auto-dismisses on next successful toggle (no Toast component, no separate dismiss action). |

**Why no Toast:** CONTEXT D-24 explicitly avoids the Toast component to keep Phase 2's component count at 6. Inline alerts inside the failed row are visually adjacent to the action that failed, which is a stronger UX signal than a corner toast anyway.

## Collapsible Behavior Contract (CONTEXT D-21, D-22, D-25)

Two separate Collapsible uses in the popup:

### Author group Collapsible (UI-03)

| Property | Value |
|----------|-------|
| Trigger content | `<chevron> {author} ({n})` — chevron is `ChevronDown` when open, `ChevronRight` when closed |
| Default open | `true` if author owns ≥1 experiment whose `scope.match` matches the active tab URL; `false` otherwise |
| Persisted | NO. User-toggled state is lost when popup closes. Re-evaluated on each open. |
| `aria-controls` | trigger has `aria-controls="author-group-{author}"`, content panel has `id="author-group-{author}"` |
| `aria-expanded` | Radix `Collapsible.Trigger` handles this automatically via `data-state="open|closed"` |
| Keyboard | `Enter` or `Space` toggles |
| Animation | Default Radix collapse + `tw-animate-css` `accordion-down`/`accordion-up` (200ms ease-out). With `prefers-reduced-motion: reduce`, animation is replaced by instant show/hide via `@media (prefers-reduced-motion: reduce) { [data-state] { animation: none !important; } }` block in `popup.css`. |

### Stack-trace Collapsible (UI-08, CONTEXT D-25)

| Property | Value |
|----------|-------|
| Trigger content | `▶ Show stack trace` (when closed) / `▼ Hide stack trace` (when open). The `▶`/`▼` is rendered as `<ChevronRight class="size-3" />` / `<ChevronDown class="size-3" />`, NOT as a Unicode arrow (consistency with author group). |
| Trigger styling | `<Button variant="ghost" size="sm" className="h-auto p-0 text-xs font-medium">` — looks like a clickable label, not a button |
| Default open | `false` (always start collapsed) |
| `aria-controls` / `aria-expanded` | same as above (Radix-managed) |
| Content panel | `<div class="mt-2 rounded border bg-muted/40 p-2 text-xs font-mono whitespace-pre">{stack}</div>` |
| Scrolling | If stack > 10 lines (~140px at `text-xs` line-height 1.4), wrap content in `<ScrollArea className="max-h-[140px]">` |
| Animation | Same as author group. |

### Primary CTA

The popup's primary CTA is the **Switch toggle**, not a button. The CTA verb+noun pair is implicit in the Switch's `aria-label`: "Toggle {experiment.name}". There is no "Save", "Apply", or "Confirm" button anywhere in Phase 2's popup — toggling the Switch IS the action and it commits immediately.

### Destructive Actions in Phase 2

There are NO destructive actions in Phase 2. Toggling an experiment OFF is reversible (user can toggle it back ON). Auto-disable is engine-driven, not user-driven. Phase 2 has no Delete, no Reset, no Wipe. Phase 3 (TWK-05 "Reset to defaults") and Phase 6 (`SHR-03` fork) are the first phases with potentially-destructive UX; they bring `Dialog` for confirmation.

### Pluralization

- `experiments` is always lowercase plural ("Add one under experiments/...")
- Author count: `andrew (1)` (no "experiment" suffix; the parens are unambiguous in context)
- Error count: `Disabled after 3 errors in 10s` (always plural — threshold is ≥3 so singular never applies)

### Locale

Phase 2 popup copy is **English-only**. No i18n infrastructure (deferred indefinitely; not a v1 requirement). The PROJECT.md Russian-language requirement statements are for spec authoring; user-facing copy is English (Andrew confirmed via existing Phase 1 strings: "No experiments yet…" is English).

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Color-blind safety | Every Badge has a text label (5 distinct labels for 5 states). Color alone is never the differentiator. |
| Screen reader: experiment toggle | `<Switch aria-label="Toggle {experiment.name}" />` — required on every Switch |
| Screen reader: collapsible groups | `Collapsible.Trigger` carries `aria-controls={id}` and `aria-expanded` (Radix-managed); `Collapsible.Content` has matching `id` |
| Screen reader: error stack expansion | Stack-trace Collapsible follows the same `aria-controls`/`aria-expanded` contract |
| Screen reader: status announcement | Badge text is read; the badge label IS the status. No separate `aria-live` region — popup is short-lived, status changes only on user toggle (same gesture user just performed). |
| Keyboard navigation | All interactive elements reachable via `Tab` in DOM order: H1 (skipped — not focusable) → Author 1 trigger → Card 1 Switch → Card 1 stack trigger (if error) → Card 2 Switch → ... → Author 2 trigger → ... |
| Focus ring | shadcn default `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2` on all interactive elements. Override forbidden. |
| Tap target minimum | Switch: 24×44px (shadcn default — meets WCAG 2.1 AA). Button (ghost "Show stack trace"): 32px tall (`size="sm"`). Collapsible author trigger: full row width × 32px tall. |
| Reduced motion | `@media (prefers-reduced-motion: reduce) { [data-state] { animation: none !important; } }` block added to `popup.css` after shadcn init |
| Contrast (WCAG AA) | shadcn `slate` palette ships with verified 4.5:1 body / 3:1 UI contrast in both light and dark modes. Do NOT modify CSS-vars in Phase 2; if a future phase introduces brand colors, re-verify contrast at that time. |

| Card hover | NONE in Phase 2 (deliberate — extension popup is not a hover-rich surface; cards aren't interactive at the card level, only the Switch and stack-trigger inside are). |

## Dark Mode Posture

**Popup follows OS `prefers-color-scheme` automatically.**

shadcn's default `slate` baseColor + cssVariables generates two `:root` blocks (`@theme inline { ... }` and `.dark { ... }`). Tailwind v4 wires the `.dark` class to `prefers-color-scheme: dark` via `@media (prefers-color-scheme: dark)` selector at the bottom of `popup.css` post-init.

**Phase 2 ships NO manual theme toggle.** Users who want forced light/dark in the popup get OS-level control, which is sufficient for v1.

**Verify on Chrome:** open Chrome on macOS with system dark mode, click extension icon → popup background flips to slate-950, Card surfaces flip to slate-900, text flips to slate-50. Verified visually during Wave 5 manual smoke per 02-RESEARCH.md §12.
