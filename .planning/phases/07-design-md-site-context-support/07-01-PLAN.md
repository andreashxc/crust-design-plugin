# Plan 07-01: Research DESIGN.md Alpha Spec And Context Shape

## Goal

Understand the current Google DESIGN.md alpha format and decide how Crust should model target-site design context without overfitting to one ya.ru file.

## Tasks

- Read the official `google-labs-code/design.md` spec and CLI behavior.
- Analyze the supplied ya.ru `DESIGN.md` example as a real-world large context file.
- Define the minimum fields Crust should understand: front matter, token maps, markdown sections, site notes, extension pitfalls.
- Decide what is validated strictly vs preserved as unknown content.
- Record risks around alpha-format churn and large-file prompt bloat.

## Acceptance Criteria

- `07-RESEARCH.md` documents the spec summary, local file shape, and large-file strategy.
- The plan clearly distinguishes Google DESIGN.md standard behavior from Crust-specific site-context extensions.
- Open questions for product/design are listed explicitly.
