# Plan 07-04: Experiment Authoring Integration

## Goal

Attach relevant design context to experiment creation/fork workflows so agents write site-consistent experiments by default.

## Tasks

- Match design context to experiment scope/URL, e.g. `ya.ru/search` maps to `design-context/ya.ru/DESIGN.md`.
- Update create/fork scripts or generated prompts to mention available context.
- Ensure prompts tell agents to use context for visual decisions, DOM safety, and known target-site pitfalls.
- Avoid changing runtime experiment behavior unless the generated code itself uses the context.
- Add tests proving context matching and prompt summary selection.

## Acceptance Criteria

- When creating a ya.ru experiment, the agent receives relevant site context automatically or via a clear command.
- If no context exists, workflows continue normally.
- Private context is never copied into experiment folders or registry output unless explicitly requested.
