# Plan 07-02: Storage Convention, Privacy, And Docs

## Goal

Make it obvious where people put private design context and make accidental publication unlikely.

## Tasks

- Finalize local directory convention, likely `design-context/<site>/DESIGN.md` and/or `.crust/design-context/<site>/DESIGN.md`.
- Keep local context ignored by git by default.
- Update onboarding and experiment authoring docs in Russian and English.
- Add vibe-coder prompts explaining that the user describes desired experiment behavior and target pages, while the agent finds relevant design context.
- Explain how public example contexts can be intentionally committed if needed.

## Acceptance Criteria

- A non-developer can add a local `DESIGN.md` for a site without guessing paths.
- Docs clearly say these files are private by default.
- No private context path is included in build artifacts by default.
