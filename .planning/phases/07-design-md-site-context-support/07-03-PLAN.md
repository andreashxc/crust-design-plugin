# Plan 07-03: Parser, Validation, And Prompt Summary

## Goal

Build a small parser/indexer that can read local design context files, validate the useful parts, and produce compact summaries for AI agents.

## Tasks

- Parse YAML front matter and markdown body.
- Validate Google DESIGN.md basics: front matter presence, token references, duplicate sections, canonical section-order warnings.
- Preserve unknown sections instead of rejecting useful site-specific notes.
- Generate a compact prompt summary with tokens, component rules, DOM landmarks, responsive notes, and extension pitfalls.
- Add tests using a representative fixture derived from the ya.ru example without committing private full context.

## Acceptance Criteria

- Parser handles a large file without excessive memory or output size.
- Invalid/broken token references produce readable warnings.
- Summary output is deterministic and small enough for agent prompts.
