<objective>
Build the reusable popup tweak renderer. After this plan, the popup has shadcn-compatible primitives and a React renderer that can display and update all six tweak types without per-experiment React code.
</objective>

<tasks>

## Task 1: Add UI Primitives

Files:

- `apps/extension/src/components/ui/input.tsx`
- `apps/extension/src/components/ui/select.tsx`
- `apps/extension/src/components/ui/slider.tsx`
- `apps/extension/src/components/ui/checkbox.tsx`
- `apps/extension/package.json`
- `pnpm-lock.yaml`

Action:

- Add shadcn-compatible primitives following existing local style.
- Install Radix deps only as needed.
- Use native color input locally rather than adding a third-party picker.

Acceptance criteria:

- Components compile with existing Tailwind v4 setup.
- No nested card patterns introduced.

Verify:

- `corepack pnpm -F @platform/extension compile`
- `corepack pnpm lint`

## Task 2: Add Popup Tweak Pure Helpers

Files:

- `apps/extension/src/popup/tweaks.ts`
- `apps/extension/src/popup/tweaks.test.ts`

Action:

- Add helpers for:
  - control write mode: immediate vs debounced.
  - value formatting for display.
  - detecting dirty values vs defaults.
  - applying a single key update to a value map.
- Cover all six tweak types with tests.

Acceptance criteria:

- Text uses 400ms debounce classification.
- Number slider uses 300ms debounce classification.
- Toggle/select/color/multi-select are immediate.

Verify:

- `corepack pnpm vitest run apps/extension/src/popup/tweaks.test.ts`

## Task 3: Implement `TweakControls`

Files:

- `apps/extension/entrypoints/popup/components/TweakControls.tsx`
- `apps/extension/entrypoints/popup/components/TweakControls.test.tsx`

Action:

- Render all six tweak types from `TweakDefinition[]`.
- Accept props:
  - `tweaks`
  - `values`
  - `errors`
  - `onChange(nextValues)`
  - `onReset()`
- Debounce text and number-slider `onChange`; immediate controls call `onChange` directly.
- Show validation messages near the relevant control.
- Show a row-level Reset button when tweaks exist.

Acceptance criteria:

- Tests can change each control and observe expected `onChange`.
- Debounced controls use fake timers.
- Reset calls `onReset`.

Verify:

- `corepack pnpm vitest run apps/extension/entrypoints/popup/components/TweakControls.test.tsx`
