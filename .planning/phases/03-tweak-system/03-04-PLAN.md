<objective>
Make content reconcile tweak-aware. After this plan, enabled experiments receive validated tweak values, invalid values prevent apply, and changing tweak values triggers cleanup -> apply without page reload.
</objective>

<tasks>

## Task 1: Add Tweak-Aware Engine Helpers

Files:

- `apps/extension/src/content/engine.ts`
- `apps/extension/src/content/tweaks.test.ts`

Action:

- Add helpers to compute a stable values key and decide whether an applied experiment needs reapply.
- Applied state should include `{ cleanup, controller, valuesKey }`.
- Tests cover unchanged values, changed values, and invalid values.

Acceptance criteria:

- Pure tests prove changed values require cleanup/reapply.

Verify:

- `corepack pnpm vitest run apps/extension/src/content/tweaks.test.ts`

## Task 2: Load And Validate Values In Content Script

Files:

- `apps/extension/entrypoints/content-isolated.content.ts`
- `apps/extension/src/content/error-isolation.test.ts`

Action:

- Read `getTweakValues(entry.id)` during reconcile.
- Merge with defaults and validate before `apply()`.
- Pass validated values as `ApplyArgs.tweaks`.
- If invalid:
  - cleanup currently applied experiment.
  - set tweak validation errors.
  - skip apply.
  - do not auto-disable as runtime failure.

Acceptance criteria:

- Existing apply args test expects non-empty tweak values when manifest declares defaults.
- Invalid values skip loader/apply.

Verify:

- `corepack pnpm vitest run apps/extension/src/content/error-isolation.test.ts apps/extension/src/content/tweaks.test.ts`

## Task 3: Broadcast On Tweak Changes

Files:

- `apps/extension/src/shared/messages.ts`
- `apps/extension/entrypoints/background.ts`
- `apps/extension/entrypoints/popup/components/ExperimentRow.tsx`

Action:

- Prefer reusing `STATE_CHANGED({tabId})` broadcast if current toggle handler shape can be reused safely.
- If a dedicated message is clearer, add `TWEAKS_CHANGED({id})` handled by SW to broadcast to matching tabs.
- Register any new SW handler at module top level.

Acceptance criteria:

- Tweak update from popup causes active content script reconcile.
- SW remains stateless.

Verify:

- `corepack pnpm vitest run apps/extension/src/background/**/*.test.ts apps/extension/entrypoints/popup/App.test.tsx`

## Task 4: Reapply On Value Key Change

Files:

- `apps/extension/entrypoints/content-isolated.content.ts`

Action:

- If experiment is already applied and wanted, compare stored `valuesKey`.
- If changed, abort/cleanup then apply with new values.
- Preserve cleanup error handling and `applied:<tabId>` writes.

Acceptance criteria:

- Tests prove cleanup called before second apply.
- `setAppliedInTab` still reflects currently applied experiments.

Verify:

- `corepack pnpm test -- --run`
