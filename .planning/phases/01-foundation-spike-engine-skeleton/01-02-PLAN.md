    Why each case (mapped to D-16 rules):
    - id regex (uppercase-only Crockford, 26 chars, no I/L/O/U) → 4 cases (valid uppercase, lowercase reject, forbidden chars reject, wrong length reject).
    - name min(1) → 1 reject case.
    - author min(1) → 1 reject case.
    - description min(1).max(280) → 1 reject case (281 char fixture).
    - scope.match min(1) → 1 reject case (empty fixture).
    - scope.regex optional → 1 accept case (with regex array).
