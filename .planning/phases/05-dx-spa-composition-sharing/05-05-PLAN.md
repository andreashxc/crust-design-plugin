<objective>
Ship tweak presets and deterministic `description.md` generation/staleness warnings, using an honest repo-file workflow for anything the extension cannot write directly.
</objective>

<threat_model>
Threats:
- Medium: preset save UI claims success without writing a repo file. Mitigation: implement CLI/dev bridge or explicit copy/download fallback with truthful status.
- Medium: preset load applies malformed values. Mitigation: validate through existing `validateTweakValues`.
- Low: description generator overwrites manual docs. Mitigation: only overwrite `generated: true`.
</threat_model>

<tasks>

## Task 1: Add Preset Load UI

Read first:

- `apps/extension/entrypoints/popup/components/TweakControls.tsx`
- `apps/extension/entrypoints/popup/components/ExperimentRow.tsx`
- `apps/extension/src/popup/tweaks.ts`

Action:

- Render preset select/menu when `entry.presets` has values.
- On preset selection:
  - validate preset values with existing tweak validator
  - persist values to `tweaks:<id>`
  - clear tweak errors
  - broadcast `TWEAKS_CHANGED`
- Show selected preset name in muted text.

Acceptance criteria:

- Component test loads a preset and asserts `chrome.storage.local.set` receives `tweaks:<id>`.
- Invalid preset values show validation error and do not send `TWEAKS_CHANGED`.

## Task 2: Add Preset Save Workflow

Read first:

- `.planning/phases/05-dx-spa-composition-sharing/05-CONTEXT.md`
- `scripts/create-experiment.ts`
- `apps/extension/README.md`

Action:

- Implement the least fragile save path:
  - preferred: `scripts/save-preset.ts <experiment-id-or-author/folder> <preset-name> '<json-values>'`
  - popup generates a copyable command when direct dev bridge is unavailable
  - if a dev bridge is implemented, it must write a real file and return the path
- Preset filename must be slugified.
- Preset JSON shape:
  - `name`
  - `values`
  - `updatedAt`

Acceptance criteria:

- `corepack pnpm save-preset andrew/tweak-demo compact '{"headline":"Hello"}'` creates `experiments/andrew/tweak-demo/presets/compact.json`.
- Popup copy command includes `corepack pnpm save-preset`.
- UI never displays `Saved` unless the save path returned a real file path.

## Task 3: Generate Description Files

Read first:

- `tools/build-experiments.ts`
- `scripts/create-experiment.ts`
- `.planning/phases/05-dx-spa-composition-sharing/05-RESEARCH.md`

Action:

- Add `scripts/generate-descriptions.ts`.
- Generate `description.md` for experiments missing it or marked `generated: true`.
- Include:
  - manifest name/author/scope/world
  - tweak table
  - helper usage detected from `experiment.ts`
  - generated timestamp
- Add `corepack pnpm generate-descriptions` script.
- Wire generation into dev/build only if it does not create noisy rewrites on every run; otherwise document as explicit command.

Acceptance criteria:

- Test or script smoke proves generated file contains `# <experiment name>`.
- Generated file contains `generated: true`.
- Manual file with `generated: false` is not overwritten.
