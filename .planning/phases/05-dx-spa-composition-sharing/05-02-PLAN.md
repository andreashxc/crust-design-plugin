<objective>
Make the content engine re-evaluate scope and reapply experiments when SPA URL changes or dev-mode experiment chunks change, without requiring extension reload or page refresh.
</objective>

<threat_model>
Threats:
- Medium: dev polling accidentally ships in production and wastes cycles. Mitigation: guard polling behind `import.meta.env.DEV` and test production flag path.
- Medium: overlapping cleanup/apply races corrupt DOM. Mitigation: single debounced reconcile scheduler with in-flight serialization.
- Low: URL patching breaks site history. Mitigation: call original history methods and emit only after successful return.
</threat_model>

<tasks>

## Task 1: Add Shared Reconcile Scheduler

Read first:

- `apps/extension/entrypoints/content-isolated.content.ts`
- `apps/extension/src/content/engine.ts`

Action:

- Create a debounced serialized scheduler used by `STATE_CHANGED`, `TWEAKS_CHANGED`, SPA URL changes, and dev registry changes.
- Ensure only one `reconcile()` runs at a time.
- If events arrive during an in-flight reconcile, run exactly one more reconcile after it finishes.
- Default debounce: 100ms.

Acceptance criteria:

- Test proves three rapid schedule calls invoke reconcile once after debounce.
- Test proves a schedule during in-flight reconcile triggers one follow-up run.
- Existing toggle and tweak tests still pass.

## Task 2: Add Engine URL Change Watcher

Read first:

- `apps/extension/src/content/helpers.ts`
- `.planning/phases/05-dx-spa-composition-sharing/05-RESEARCH.md`

Action:

- Add `apps/extension/src/content/url-change.ts`.
- Patch `history.pushState` and `history.replaceState` once per content script.
- Listen to `popstate` and `hashchange`.
- Emit only when `location.href` changes.
- Wire watcher to the reconcile scheduler in `content-isolated.content.ts`.

Acceptance criteria:

- Unit test covers `pushState`, `replaceState`, `popstate`, and `hashchange`.
- Unit test proves no callback when `pushState` keeps the same URL.
- Content script cleanup does not leave duplicate patches when initialized once.

## Task 3: Add Dev Registry Change Reapply

Read first:

- `tools/build-experiments.ts`
- `apps/extension/entrypoints/content-isolated.content.ts`
- `packages/experiment-sdk/src/index.ts`

Action:

- Add `apps/extension/src/content/dev-refresh.ts`.
- In dev mode only, poll `chrome.runtime.getURL('registry.json')` every 750ms.
- Compute a signature from enabled/scope-matching entries using `id`, `chunkPath`, and `sourceSignature`.
- When signature changes, schedule reconcile.
- Extend applied experiment state to track `moduleKey`/`chunkPath`; reapply when module key changes even if tweak values did not.

Acceptance criteria:

- Test proves changing `chunkPath` for an enabled matching experiment triggers cleanup/apply.
- Test proves no polling starts when dev flag is false.
- Manual smoke checklist includes `<2 seconds` dev reapply.
