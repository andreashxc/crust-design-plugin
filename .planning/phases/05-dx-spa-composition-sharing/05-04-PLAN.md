<objective>
Add compact popup search, reorder controls, and Open in Cursor affordances without changing the Crust visual style.
</objective>

<threat_model>
Threats:
- Low: `cursor://` links may fail on machines without Cursor handler. Mitigation: fallback copies path and shows `Path copied`.
- Low: drag reorder can be inaccessible. Mitigation: up/down buttons are mandatory fallback.
</threat_model>

<tasks>

## Task 1: Add Search Filtering

Read first:

- `apps/extension/src/popup/grouping.ts`
- `apps/extension/entrypoints/popup/App.tsx`
- `.planning/phases/05-dx-spa-composition-sharing/05-UI-SPEC.md`

Action:

- Add search query state to popup.
- Filter after active-tab scope matching.
- Match lowercase `name`, `author`, and `description`.
- Add a compact shadcn-style search input below the header with placeholder `Search experiments`.
- Add clear button when query is non-empty.

Acceptance criteria:

- Component test: query `helper` shows Helper demo and hides Smoke pink.
- Component test: query `andrew` keeps all Andrew rows.
- Component test: no matches shows `No matches`.
- Page-scope empty state still shows `No experiments match this page`.

## Task 2: Add Reorder Controls

Read first:

- `apps/extension/entrypoints/popup/components/ExperimentRow.tsx`
- `apps/extension/src/shared/storage.ts`
- `apps/extension/src/shared/messages.ts`

Action:

- Add compact grip icon to experiment rows.
- Implement pointer drag-and-drop reordering within the visible experiment list.
- Add accessible up/down icon buttons.
- Persist updated order via `setExperimentOrder`.
- Broadcast state change after order update so content re-applies.
- Keep controls visually low-contrast until hover/focus.

Acceptance criteria:

- Component test drags one experiment row below another and asserts storage receives reordered ids.
- Component test clicks `Move Smoke pink down` and asserts storage receives reordered ids.
- Component test asserts `STATE_CHANGED` or equivalent broadcast is sent after reorder.
- Buttons have aria labels containing `Move`.

## Task 3: Add Open in Cursor Action

Read first:

- `packages/experiment-sdk/src/index.ts`
- `apps/extension/entrypoints/popup/components/ExperimentRow.tsx`

Action:

- Show `Open in Cursor` icon button only if `entry.sourceDir` exists.
- Try opening `cursor://file/{encodedAbsolutePath}`.
- If `chrome.tabs.create` or URL open fails, copy `sourceDir` to clipboard if available and show `Path copied`.

Acceptance criteria:

- Component test: row with no `sourceDir` has no Open in Cursor button.
- Component test: row with `sourceDir` shows button labelled `Open in Cursor`.
- Fallback state renders `Path copied`.
