<objective>
Extend registry/build metadata so Phase 5 runtime and popup features have one source of truth for dev source paths, source signatures, presets, and description freshness.
</objective>

<threat_model>
Threats:
- Medium: exposing arbitrary local paths in production registry. Mitigation: emit `sourceDir` only in dev/local builds or ensure it is non-secret project-relative metadata.
- Medium: malformed preset JSON could inject invalid tweak values. Mitigation: validate preset values with existing tweak validation before load/apply.
- Low: generated descriptions could overwrite hand-written docs. Mitigation: only overwrite files marked as generated.
</threat_model>

<tasks>

## Task 1: Add Additive Registry Metadata

Read first:

- `packages/experiment-sdk/src/index.ts`
- `tools/build-experiments.ts`
- `.planning/phases/05-dx-spa-composition-sharing/05-CONTEXT.md`

Action:

- Extend `RegistryEntry` with optional fields:
  - `sourceDir?: string`
  - `sourceSignature?: string`
  - `presets?: Array<{ name: string; path: string; values: Record<string, unknown> }>`
  - `descriptionStatus?: 'missing' | 'fresh' | 'stale' | 'manual'`
- Update build registry emission to fill these fields.
- Preserve all existing required fields and keep old consumers valid.

Acceptance criteria:

- `packages/experiment-sdk/src/index.ts` contains `sourceDir?: string`.
- `packages/experiment-sdk/src/index.ts` contains `descriptionStatus?: 'missing' | 'fresh' | 'stale' | 'manual'`.
- `tools/build-experiments.test.ts` asserts registry entries still include the original required fields.
- `corepack pnpm vitest run tools/build-experiments.test.ts packages/experiment-sdk/src/manifest.test.ts` exits 0.

## Task 2: Discover Preset Files

Read first:

- `tools/build-experiments.ts`
- `packages/experiment-sdk/src/index.ts`
- `apps/extension/src/popup/tweaks.ts`

Action:

- Discover `experiments/<author>/<folder>/presets/*.json`.
- Parse each preset as `{ name?: string, values: Record<string, unknown> }` or a plain `Record<string, unknown>` values map.
- Emit normalized preset objects in `RegistryEntry.presets`.
- Validate preset values against the experiment's tweak definitions when possible; invalid presets fail build with file path and issue message.

Acceptance criteria:

- A fixture preset at `experiments/andrew/smoke/presets/default.json` appears in emitted registry as `presets[0].path`.
- Invalid preset values produce a `Manifest validation failed`-style build error that includes `presets/default.json`.
- Existing experiments without presets emit `presets: []` or omit `presets` consistently.

## Task 3: Compute Description Freshness

Read first:

- `tools/build-experiments.ts`
- `.planning/phases/05-dx-spa-composition-sharing/05-RESEARCH.md`

Action:

- Add deterministic description freshness detection:
  - `missing` when `description.md` is absent.
  - `fresh` when `description.md` exists and is newer than `manifest.json` and `experiment.ts`.
  - `stale` when `manifest.json` or `experiment.ts` is newer.
  - `manual` when description frontmatter includes `generated: false`.
- Do not generate descriptions in this task; only emit status.

Acceptance criteria:

- `tools/build-experiments.test.ts` covers `missing`, `fresh`, `stale`, and `manual`.
- Registry output includes `descriptionStatus`.
