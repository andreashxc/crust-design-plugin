<objective>
Add ordered multi-experiment composition, self-mutation-safe DOM observation, and toolbar badge updates for applied experiment count.
</objective>

<threat_model>
Threats:
- Medium: order storage can reference deleted experiments. Mitigation: sorting helper ignores unknown ids and appends new ids.
- Medium: MutationObserver can loop on extension DOM. Mitigation: ignore mutations inside `[data-exp-id]`.
- Low: badge reveals count on unrelated tabs. Mitigation: update badge per `tabId` from applied state and clear on tab removal.
</threat_model>

<tasks>

## Task 1: Add Experiment Order Storage

Read first:

- `apps/extension/src/shared/storage.ts`
- `apps/extension/src/shared/storage.test.ts`

Action:

- Add helpers:
  - `getExperimentOrder(): Promise<string[]>`
  - `setExperimentOrder(ids: string[]): Promise<void>`
  - `appendExperimentOrder(id: string): Promise<void>`
  - `sortRegistryByOrder(entries, order): RegistryEntry[]`
- Store under `experiment_order` in `chrome.storage.local`.
- Unknown ids in storage must not break sorting.

Acceptance criteria:

- Storage tests cover empty/malformed order returning `[]`.
- Sorting test covers ordered ids first and unknown registry entries appended in registry order.
- `appendExperimentOrder` does not duplicate ids.

## Task 2: Apply Experiments in Stored Order

Read first:

- `apps/extension/entrypoints/content-isolated.content.ts`
- `apps/extension/src/content/engine.ts`

Action:

- Load experiment order during reconcile.
- Sort `wantOn` by stored order before apply.
- When an experiment is toggled on and missing from order, append it.
- Ensure reordering triggers cleanup/apply in the new order on next reconcile.

Acceptance criteria:

- Content test proves two enabled experiments apply in stored order.
- Content test proves changing order causes both matching experiments to reapply in new order.

## Task 3: Add Self-Mutation-Safe Observer

Read first:

- `apps/extension/src/content/helpers.ts`
- `apps/extension/entrypoints/content-isolated.content.ts`

Action:

- Add shared observer utility that watches page DOM changes and schedules reconcile for dynamic SPA DOM readiness.
- Ignore mutation targets and added nodes that are inside `[data-exp-id]`.
- Debounce observer-triggered reconcile using the scheduler from 05-02.

Acceptance criteria:

- Test proves adding a normal page node schedules reconcile.
- Test proves adding a node under `[data-exp-id]` does not schedule reconcile.
- Test proves rapid mutations debounce to one schedule call.

## Task 4: Add Toolbar Badge Updates

Read first:

- `apps/extension/src/background/action-icon.ts`
- `apps/extension/entrypoints/background.ts`
- `apps/extension/src/shared/messages.ts`

Action:

- Add typed message `APPLIED_COUNT_CHANGED(data: { tabId: number; count: number })`.
- Content script sends it after applied ids are written.
- Background sets badge text:
  - `''` when count is 0
  - `String(count)` for 1-99
  - `99+` for 100+
- Clear badge when tab closes.

Acceptance criteria:

- Background unit test proves count 0 clears badge.
- Background unit test proves count 5 sets text `5`.
- Background unit test proves count 120 sets text `99+`.
